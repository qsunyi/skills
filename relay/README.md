# relay

一个用于生成、更新、补全、重写和查看会话 / 项目交接文档的 skill。

`relay` 的目标不是做普通聊天摘要，而是把当前任务状态压缩成一份**可续跑、可验证、低重复**的接力文档，供下一个 session、下一个 agent 或未来的自己继续执行。

## 适用场景

适用于以下类型的请求：

- 生成 handoff / 交接文档
- 在 `/compact` 前整理续跑说明
- 长时间暂停前保存当前进展
- 多文件修改后的接力说明
- 浏览器自动化、复杂排障、长链路调试后的状态整理
- 更新已有 relay / handoff 文件
- Relay Mesh 工作流中的主线 / 分支接力整理

常见触发说法包括：

- “帮我做个 relay”
- “生成一个 handoff / 交接文档”
- “写个 /compact 前的续跑说明”
- “更新一下 relay 文件”
- “帮我整理当前进展，给下个 session 用”

## 不适用场景

以下情况默认不直接触发 relay：

- 普通口头总结
- 阶段性进展汇报
- 单纯继续执行任务
- 普通任务清单管理

例如：

- “继续”
- “接着做”
- “总结一下”
- “说说现在进展”

这些表达可能只是要一个口头总结，不等于要落盘生成交接文档。

## 核心原则

`relay` 的核心目标是：

1. 让下一个接手者快速知道用户目标是什么；
2. 明确哪些事实已经验证，哪些只是猜测；
3. 说明已经试过什么、改过什么、为什么这样做；
4. 降低重复踩坑和重复探索的成本。

因此它应优先保留：

- 下一步建议
- 当前 blocker
- 已确认事实
- 待验证假设
- 关键文件路径
- 关键命令摘要
- 已踩过的坑

## 与其他 skill 的区别

- `relay`：负责上下文与知识接力
- `ledger`：负责任务与执行状态台账

如果用户需要的是交接上下文，优先使用 `relay`；
如果用户需要的是追踪任务状态，优先使用 `ledger`。
